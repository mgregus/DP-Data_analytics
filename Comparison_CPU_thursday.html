Vyhodnotenie vkonovch dopadov

vod

V rmci diplomovej prce sme vytvorili sadu CLI skriptov umoujcich pouvateovi aplikova a deaplikova vybran skupiny pravidiel na zklade rznych atribtov pravidla a jeho hodnoty. Nsledne sme otestovali pouitie nstroja na redukciu pouitej deteknej sady formou deaktivcie pravidiel, ktor sme nepovaovali za relevantn pre testovan infratruktru z hadiska bezpenostnej zvanosti upozornen vygenerovanmi pravidlom alebo z hadiska jeho relevantnosti vzhadom na typy zariaden a pouit technolgie v chrnenej infratruktre.

Celkovo sa nm pouitm vytvorenej sady CLI skriptov podarilo zredukova detekn sadu o 5 957 z pvodnch 30 793 pravidiel na 24 836 pravidiel, o predstavuje redukciu pouitej deteknej sady o takmer 20% pravidiel. Priom ako sme ukzali v sady rmci experimentov zameranej na otestovanie dopadov redukcie deteknej sady na mnostvo a charakter vygenerovanch upozornen tto redukcia nemala iadne negatvne dopady na schopnos bezpenostnch analytikov vyhodnocujcich vygenerovan upozornenia identifikova kodliv sieov premvku v sieti.

V rmci tejto sady experimentov sme sa teda zamerali na vyhodnotenie dopadov redukcie pouitej deteknej sady pravidiel na redukciu potrebnch vpotovch zdrojov systmom IDS.

Vykonan experimenty

Rovnako ako v prpade testovania dopadov redukcie deteknej sady aj v tomto prpade sme vykonali testovanie v rmci navrhnutej a implementovanej testovacej ifratruktry s pouitm pvodnho datasetu. Experiment bol vykonan s nasledovnmi parametrami:

  • Rchlos prehrvanie dt: 100 mbps
  • Nastavenie N (poet analyzovanch paketov z kadho toku): 100
  • Poet opakovan experimentu: 10 X

Na monitorovanie vpotovch poiadaviek systmu IDS poas testovania sme pouili nstroj Psrecord. Vstupu z tohto programu vo formte txt boli predmetom vkonovej analzi. Aby sme dosiahli ist mieru ustlenosti vsledkov meranie pre kad testovac scenr (Pondelok a Piatok) sme zopakovali 10 X pre oba testovacie prpady teda pvodn aj redukovan sada pravidiel. V rmci analzi pre jednotliv testovacie scenre vyhodnocujeme teda vsledne kumulatvne ako shrn zznam z jednotlivch testovan.

V nasledujcej asti porovnvame vyuitie vpotovch prostriedkov systmom IDS poas 4. tesovacieho scenru (tvrtok) pri pouit pvodnej deteknej (df_original_disable) sady a redukovanej deteknej sady (df_modified_disable).

Spracovanie dt

Vstupy z programu Psrecord pre jednotliv testovacie experimenty vo formte txt sme natali z disku. A vytvorili sme dve dtov sady obsahujce zznamy o spotrebe vpotovch zdrojov poas 10 vykonan experimentu pre vybran scenr pri aplikovan:

  1. Pvodnej sady pravidiel - (df_original_disable)
  2. Redukovanej sady pravidiel - (df_modified_disable)

Pod tmto blokom je uveden formt analyzovanch dt. Ako mono vidie zznami obsahuj diskrtne hodnoty vyuitia CPU a pamte RAM systmom IDS na testovacej intancii Flowmon Collector VA zaznamenan v ase programom Psrecord.

def concatenate_files_in_folder(folder_path, file_extension=".txt"):    
    # List all files in the specified folder
    files = os.listdir(folder_path)

    datafiles = []
    
    # Filter files by the specified extension
    data_files = [f for f in files if f.endswith(file_extension)]

    # Initialize an empty list to store DataFrames
    dataframes = []

    # Read each file into a DataFrame and add it to the list
    for file in data_files:
        file_path = os.path.join(folder_path, file)  # Full path to the file
        df = pd.read_csv(file_path, delim_whitespace=True, comment='#')  # Read the file
        dataframes.append(df)

    # Concatenate all DataFrames into one
    concatenated_df = pd.concat(dataframes, ignore_index=True)  # Combine DataFrames with new indexing
    
    return concatenated_df

df_original_disable = concatenate_files_in_folder(folder_path_original, ".txt")  
df_modified_disable = concatenate_files_in_folder(folder_path_modified, ".txt")
